# Codecov Setup Reference

Use this reference when creating or revising `codecov.yml` or GitHub Actions upload wiring.

## Source Links

- Codecov API reference: `https://docs.codecov.com/reference`
- Codecov configuration reference: `https://docs.codecov.com/docs/codecov-yaml`
- Codecov GitHub Action: `https://github.com/codecov/codecov-action`
- Codecov status checks: `https://docs.codecov.com/docs/commit-status`
- Codecov flags: `https://docs.codecov.com/docs/flags`
- Codecov components: `https://docs.codecov.com/docs/components`

Check current docs before adding an option that is not already present in this repository or generated by the helper.

## Configuration Rules

- Keep project and patch status targets meaningful. Do not lower targets just to make a failing PR green.
- Use flags when separate test suites or languages upload separate reports.
- Use carryforward flags only when a flag is intentionally absent from some commits, such as matrix-specific or path-specific reports.
- Keep ignored paths narrow and explain why generated, vendored, or fixture paths should not affect coverage.
- Set `threshold` intentionally. A nonzero threshold permits small drops; it should match team policy, not hide regressions.
- Use components when a repository needs durable coverage ownership boundaries, such as packages, apps, or languages.

## GitHub Actions Upload Pattern

Prefer OIDC for GitHub Actions when the repository is configured for Codecov's OIDC/tokenless upload path. Otherwise use `CODECOV_TOKEN` as a GitHub secret and pass it through the action's `token` input.

Use separate upload steps for coverage and test results when both are sent to Codecov:

```yaml
- name: Upload coverage to Codecov
  uses: codecov/codecov-action@v7
  with:
    use_oidc: true
    files: coverage/python.xml
    flags: python
    name: pytest-coverage-python
    report_type: coverage
    fail_ci_if_error: true
    disable_search: true
    verbose: true

- name: Upload test results to Codecov
  uses: codecov/codecov-action@v7
  with:
    use_oidc: true
    files: test-report.junit.xml
    flags: python
    name: pytest-junit-python
    report_type: test_results
    fail_ci_if_error: true
    disable_search: true
    verbose: true
```

Pin third-party actions according to the repository's workflow policy. In this user's repos, many workflow actions are pinned to full SHAs with comments naming the resolved version.
